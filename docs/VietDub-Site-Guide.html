1Tổng quan & stack miễn phí

Website cho phép khách tự mua VietDub bằng VietQR: khi khách quét mã và chuyển khoản, hệ thống tự xác nhận, lưu đơn vào Google Sheet, rồi gửi link tải + link nhóm Zalo về email và hiển thị ngay trên trang cảm ơn. Toàn bộ vận hành không tốn phí cố định.

Thành phầnDịch vụChi phí
Web + API webhookVercel Hobby (Next.js + serverless)Miễn phí
Tên miềnvietdub.vercel.app hoặc domain riêngMiễn phí (subdomain)
Lưu đơn hàngGoogle SheetsMiễn phí
Thanh toánPayOS (VietQR + webhook)Miễn phí, không phí tháng
File cài đặtGitHub Releases (.exe trên VietDubPublic)Miễn phí
Gửi emailGmail SMTP (app password)Miễn phí
Mô hình tin tưởng (honor-system)

File .exe nằm trên repo public nên ai có link đều tải được. Website chặn người chưa trả tiền thấy link, nhưng không chống được việc người mua chia sẻ lại file. Muốn chống sao chép thật cần thêm license key trong app VietDub (một hạng mục riêng, lớn hơn).

2Luồng hoạt động end-to-end

1
Khách bấm “Mua” → điền emailModal thu email (+ tên, SĐT tuỳ chọn).
2
POST /api/create-paymentSinh mã đơn, ghi đơn PENDING vào Google Sheet, tạo link PayOS (giá tính lại ở server theo khuyến mãi).
3
Chuyển sang trang thanh toán PayOSKhách quét VietQR và chuyển khoản.
4
PayOS gọi POST /api/webhookXác thực chữ ký PayOS → cập nhật đơn thành PAID → gửi email link tải + Zalo (idempotent, không gửi trùng).
5
Khách quay về /success?orderCode=…Trang tự hỏi /api/order-status tới khi PAID rồi hiện nút Tải + Zalo.
Webhook không chạy trên localhost

PayOS ở ngoài Internet không gọi vào máy bạn được. Bước 4 chỉ test đầy đủ sau khi deploy lên Vercel (lúc đó mới có URL công khai cho webhook).

4Biến môi trường & quy tắc bảo mật

Có 3 nơi độc lập. Secret không bao giờ lên GitHub:

NơiChứa gìCó secret?
GitHub (repo)Chỉ code❌ Tuyệt đối không
.env.local (máy bạn)Secret bản local✅ — chạy npm run dev
Vercel → Environment VariablesSecret bản production✅ — web thật chạy

Trong code, process.env.X lấy giá trị từ môi trường đang chạy: máy bạn → .env.local; trên Vercel → các ô bạn gõ trong dashboard. Vì vậy .env.local bị gitignore mà web vẫn chạy (Vercel không đọc file này).

5Thiết lập PayOS

  1. Đăng ký tại my.payos.vn, tạo Kênh thanh toán (liên kết tài khoản ngân hàng).
  2. Vào kênh → mục API, lấy PAYOS_CLIENT_ID, PAYOS_API_KEY, PAYOS_CHECKSUM_KEY.
  3. Sau khi deploy, đặt Webhook URL:
    https://<domain>/api/webhook?token=<WEBHOOK_TOKEN> rồi bấm Kiểm tra.

6Thiết lập Google Sheets

  1. Tạo Google Sheet trống. Lấy GOOGLE_SHEET_ID trong URL (docs.google.com/spreadsheets/d/SHEET_ID/edit). Tab Orders và Settings tự tạo ở lần ghi đầu.
  2. Vào Google Cloud Console → tạo project → bật Google Sheets API.
  3. Credentials → Service account → tạo key JSON. Lấy client_email → GOOGLE_SERVICE_ACCOUNT_EMAIL; private_key → GOOGLE_PRIVATE_KEY.
  4. Chia sẻ Google Sheet cho client_email đó, quyền Editor.
Hay quên: bước chia sẻ Sheet

Thiếu bước chia sẻ cho email service account sẽ lỗi 403. Đây là lỗi phổ biến nhất khi cấu hình.

7Thiết lập email (Gmail) — tuỳ chọn

  1. Bật Xác minh 2 bước cho Gmail.
  2. Tạo App password (myaccount.google.com/apppasswords) — chuỗi 16 ký tự.
  3. Điền SMTP_USER = email Gmail, SMTP_PASS = app password.
Bỏ trống cũng không sao

Không điền SMTP_USER/SMTP_PASS thì hệ thống bỏ qua gửi email — khách vẫn nhận link tải ngay trên trang /success.

8Giá, khuyến mãi & trang /admin

Giá/khuyến mãi/link không cố định trong code: chúng nằm trong tab Settings của Google Sheet và sửa được mà không cần deploy lại.

Cách giảm giá hoạt động

  • Khi now < PROMO_ENDS_AT và PROMO_PRICE < BASE_PRICE: trang hiện giá khuyến mãi, gạch ngang giá gốc, badge −X% và đồng hồ đếm ngược.
  • Hết hạn: giá tự động về BASE_PRICE.
  • Giá khách trả luôn được tính lại ở server — không sửa được từ trình duyệt.

CáchThao tácDeploy lại?

Sửa thẳng Google SheetĐổi ô trong tab Settings❌ Không
Đổi biến môi trường VercelSửa BASE_PRICE… rồi Redeploy✅ Có

Trang /admin (vào bằng ADMIN_PASSWORD) cho sửa giá gốc / giá KM / hạn khuyến mãi / tên SP / link tải / link Zalo / link demo, và xem nhanh số đơn · doanh thu · đơn gần đây. Thay đổi áp dụng trong ~30 giây.

9Chạy thử trên máy (local)

cd D:\AgenticAI\vietdub-site
cp .env.example .env.local   # rồi điền secret
npm install                  # lần đầu (đã cài sẵn)
npm run dev                  # http://localhost:3000
  • Trang chủ hiện ngay kể cả chưa điền secret (giá về mặc định).
  • Nút “Mua” cần khoá PayOS + Google Sheet mới hoạt động.
  • Webhook tự xác nhận chỉ test được sau khi deploy (xem mục 2).

10Deploy lên Vercel

  1. Push code lên GitHub (đã có: github.com/kaisye/vietdub-site).
  2. vercel.com → Add New → Project → chọn repo → Import.
  3. Settings → Environment Variables: thêm tất cả biến (mục 4) với giá trị thật. Đổi NEXT_PUBLIC_SITE_URL thành domain Vercel.
  4. Deploy → nhận domain https://<tên>.vercel.app.
  5. Cập nhật NEXT_PUBLIC_SITE_URL = domain đó → Redeploy.
  6. Vào PayOS đặt Webhook URL trỏ về …/api/webhook?token=….
Tự động deploy mỗi lần push

Sau khi liên kết, mỗi git push lên nhánh main sẽ tự kích hoạt Vercel build lại bản mới. Không cần thao tác thủ công.

12Checklist phát hành

  • ☐ PayOS: 3 khoá API đã nhập, kênh thanh toán hoạt động
  • ☐ Google Sheet đã chia sẻ cho service account (quyền Editor)
  • ☐ Tất cả biến môi trường đã nhập trên Vercel
  • ☐ NEXT_PUBLIC_SITE_URL = domain thật + đã Redeploy
  • ☐ Webhook URL ở PayOS trỏ đúng …/api/webhook?token=… và đã “Kiểm tra” OK
  • ☐ ADMIN_PASSWORD đặt mật khẩu mạnh
  • ☐ DOWNLOAD_URL trỏ đúng .exe mới nhất
  • ☐ ZALO_GROUP_URL đúng link mời nhóm
  • ☐ Mua thử 1 đơn thật để xác nhận toàn bộ luồng
